# Grokestrator — working agreement

Native macOS (hybrid host+client) + iOS (client) SwiftUI control plane over `grok`
coding agents, spoken to over **ACP** (newline-delimited JSON-RPC 2.0 to
`grok agent stdio`). Design lives in `design/`; the AI-native thesis in `~/dev/AI`.

## Workflow

- **At the start of every new slice on clean `main`, check open GitHub issues**
  (`gh issue list`). If any is serious enough to deserve fixing before the planned
  work — a crash, data loss, a broken core flow, a regression — **say so before
  starting**, and propose fixing it first. Don't silently build new features on top
  of a known serious bug. Close/annotate issues that a merged (or in-flight) PR
  resolves, citing the PR.
- **One slice = one branch off `main` = one PR.** Never start a slice on the
  current feature branch; branch fresh from `main`. Keep PRs small and focused.
- Commit/push only when asked (or per an explicit standing request); branch first
  if on `main`.

## Build & verify

- The `.xcodeproj` is generated by **XcodeGen** and gitignored. After pulling,
  switching branches, or adding/removing files: `xcodegen generate`.
- **Build both targets** before handing back: `GrokestratorMac` (macOS) and
  `GrokestratoriOS` (iOS Simulator). CI's macos runner enforces Swift 6 actor
  isolation *stricter* than local Xcode — for release-sensitive changes, dry-run
  the Release workflow before tagging.
- **Self-verify UI/behavior in a standalone harness** before claiming it works
  (e.g. an off-screen `ImageRenderer` snapshot, a headless AppKit window, or
  driving `grok agent stdio` directly). Don't ship untested UI guesses.

## Architecture notes

- `ConversationViewModel` (`@MainActor @Observable`) bridges the broadcast
  subscription to SwiftUI. The host is the source of truth; every device subscribes
  to the same per-Connection stream, so **anything that must reach other devices has
  to round-trip through the host** as a broadcast update — not just a local mutation
  (e.g. dismissing an answered question, cancelling a turn).
